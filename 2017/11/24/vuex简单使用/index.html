Contents

VUE及社区的文档都非常友好，先打开 VUEX文档：Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式。它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。

如果您不打算开发大型单页应用，使用 Vuex 可能是繁琐冗余的。

还是那句炸天的话：

您自会知道什么时候需要它

通常管用到vuex叫做放到store里面，store就是小卖部、小仓库，这篇文章是按照我写项目的逻辑来的，你需要把它看成一个整体。

state

我会把state看作在.vue文件中用得最多的data()，这里定义了一些数据的默认状态，但是state是单一状态树，是一个全局的唯一数据源，可以把它看作本地存储，在全局使用到。

可以在某个组件中通过computed获取state中的状态

mutations

把它看作放置函数进行状态更改的地方，并且它会接受 state 作为第一个参数

Getter

可看作 store 的计算属性，假如你在页面中用到了state里的状态，并且需要对该状态进行过滤并计算，还有还几个页面多处也用到了这个过滤计算，那么你在store里像computed()一样写getter，它的返回值会根据它的依赖被缓存起来，且只有当它的依赖值发生了改变才会被重新计算。

这样 getters 会暴露为 store.getters 对象

Module

将 store 分割成模块，每个模块拥有自己的 state、mutation、action、getter、甚至是嵌套子模块

Action

运用同mutations一样，会自动获得一个默认参数context，即接受一个与 store 实例具有相同方法和属性的 context 对象，文档提到 action 和 mutations 的区别主要在于action可异步操作

事实上在 vuex 里面 actions 只是一个架构性的概念，并不是必须的，说到底只是一个函数，你在里面想干嘛都可以，只要最后触发 mutation 就行，参考这篇文章
