静态布局

最原始的布局方式，就是你把窗口缩小，页面及字体不做变化，网页最外层容器有固定的大小（例如：min-width:1000px;），页面所有的内容以该容器为准，超出宽高的部分滚动。

所以当pc端是这种布局，但是移动端也需要查看时，pc页面会显得很丑，真的不合适啊，通常又会给移动端设计一套，通常会限制最大的宽度，超过了则以最大宽度居中显示，即楼下👇所说的固定页面宽度的做法，用另一个域名呈现，一般管它叫做m站。

固定页面宽度

直接固定页面的宽度，超出部分就留白，这样的做法对于前端来说十分友好，不用绞尽脑汁想着适配，但是这样的做法对用户来说十分不好看且不友好，在大屏下会显得页面特别小。

百分比布局（流式布局）

顾名思义，百分比。随着屏幕的改变，页面的布局像液体一样流动，没有发生大的变化。

响应式布局


- 原理
通过查询screen的宽度来指定某个宽度区间的网页布局 (CSS3 media query)

即一个网站兼容多个终端，适用于中小门户。
- 原因（相对公司）
1. 移动端屏幕越来越大
2. 提高用户体验
3. 降低开发和运营成本，方便数据维护
4. 不同的网址不利于搜索引擎优化收录，影响网站搜索排名

- 栗子
@media screen and (max-width:480px){ }
@media screen and (min-width: 992px) and (max-width: 1200px){ }


- 背景图片自适应 图片失真
有时会遇到适配时在小屏上只展示图片缩略图，大屏才展示大图
backgroundsize设置 为原图片的一半 50%
backgroundposition 设置 为本应该设置的一半

rem布局

rem 在webapp下作用尤其突出

  • 单位
    px 是一个绝对单位，特别死
    em（font size of the element）相对于父元素的字体大小
    rem (font size of the root element)单位是基于根元素< html >的字体大小，宽高内容都能自适应
  • JS配合less去动态计算根元素的font-size，这样的好处是所有设备分辨率都能兼容适配

此时您的less将生成一份在设备屏幕清单里对应html的font-size的css文件。

viewport缩放

在移动设备当中，浏览器和网页之间还有一层虚拟的容器，叫 viewport，只在移动设备才有，默认就存在，虚拟的容器用来承载网页的，而且主流的设备当中的viewport宽度默认是980px,可以缩放，可以设置尺寸，是否运行用户自行缩放。

  • 功能
    承载网页 可以缩放 可以设置是否允许用户缩放
  • 栗子
    <meta name="viewport" content="width=320,maximum-scale=1.3,user-scalable=no">
    <meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=0">

使用viewport和流式布局达到移动端的适配

    • width 当前viewport的宽度
    • initial-scale 视口的默认的缩放比
    • user-scalable 是否运行用户自行缩放

如果没有指定默认的缩放值，移动设备上的浏览器默认自己必须要让所有的网站都正常显示，所以会自动计算这个缩放值initial-scale，以达到当前页面不会出现横向滚动条(或者说viewport的宽度就是屏幕的宽度)的目的，此时会造成因为移动设备的viewport太窄，pc页面在移动设备挤作一团。

如果给viewport设一个值，比如980，这样的话即使是那些为桌面设计的网站也能在移动浏览器上正常显示了。

此时我们按照 PPK大神的理论：（详细可以参考这篇文章）

  1. 管浏览器默认的视口叫做 layout viewport，此宽度可以使用document.documentElement.clientWidth获取；
  2. 把浏览器可视区域的大小叫做 visual viewport，可以通window.innerWidth来获取;
  3. 还有完美适配所有移动设备的理想viewport，叫做 ideal viewport，它并没有一个固定的尺寸，不同的设备拥有有不同的ideal viewport。
    • visual viewport宽度 = ideal viewport宽度 / 当前缩放值
    • 当前缩放值 = ideal viewport宽度 / visual viewport宽度

由于不知道这个设备的理想宽度是多少，那么用device-width这个特殊值（width = device-width即让当前viewport的宽度等于设备的宽度）就行了，内容布局自适应，同时initial-scale=1也有把viewport的宽度设为理想宽度的作用。
